# Core Web Vitals + Image Audit — May 2026

## Scope

Routes audited: `/`, `/stay`, `/day-tour`, `/fun`, `/build`, `/see`.

Primary device model: mobile viewport `390x844`, matching the Facebook Messenger mobile traffic path. Local production verification used `next build` + `next start`; local dev verification used `npm run dev` as requested.

PageSpeed Insights note: no production URL was present in the repo/docs. A localhost PageSpeed API run was attempted at `https://www.googleapis.com/pagespeedonline/v5/runPagespeed?url=http%3A%2F%2Flocalhost%3A3000%2F&strategy=mobile&category=performance`, but PSI cannot reliably audit local-only URLs and returned `429 Too Many Requests`. Local Lighthouse was used for the repeatable metric baseline.

## Changes Made

- Converted the local 3.5 MB PNG logo to optimized WebP/AVIF and routed public logo usage through `next/image`.
- Downloaded current public Unsplash assets into `public/uploads/originals/`.
- Generated optimized WebP and AVIF variants in `public/uploads/optimized/`.
- Replaced public raw `<img>` usage with `components/public/OptimizedImage.tsx`.
- Replaced CSS hero background images with priority `next/image` hero images, `sizes="100vw"`, and `fetchPriority="high"`.
- Added explicit `width`/`height` for card, modal, logo, and below-fold images.
- Added `next.config.ts` image `deviceSizes`, `imageSizes`, and `formats: ['image/avif', 'image/webp']`.
- Reduced unused Cormorant Garamond font weights while keeping `next/font` and `display: 'swap'`.
- Disabled delayed mobile hero text animations so mobile LCP is not held behind visual reveal delays.

## Acceptance Status

Passed:

- Homepage total transfer reduced by more than 40%.
- CLS stayed below `0.1`.
- Optimized images are local under `public/uploads/`.
- Public above-the-fold hero images use priority `next/image` plus `fetchPriority="high"`.
- Fonts are still loaded through `next/font`; no FOUT/FOIT regressions were visible in screenshots.

Not fully proven in this environment:

- PageSpeed Insights mobile LCP `< 2.5 s`, because no production URL was available and PSI could not audit localhost. Local production Lighthouse still reports simulated mobile LCP above target on most pages, despite observed Chrome LCP being much faster locally.
